1) Visao geral
================================================================

O projeto esta com as 6 fases do plano implementadas em nivel funcional.
O foco atual esta em robustez, cobertura de testes, retriable email flows e
melhorias de operacao (bulk actions e refinamentos de UX/processo).

3) Como testar (roteiro rapido para dev local)
================================================================

Pre-requisitos:
- Docker rodando.
- Variaveis de ambiente locais configuradas (.env).

Setup rapido:
1. docker compose up -d db
2. npm run db:push
3. npm run db:seed:fast
4. npm run dev

Sugestao de contas de teste:
- admin@ufba.br
- professor@ufba.br
- student@ufba.br

Fluxo minimo recomendado de validacao manual:
- Fase 1:
  Admin importa planejamento e notifica professor.
  Professor cria/ajusta projeto, assina e submete.
- Fase 2:
  Admin aprova/rejeita e aciona fluxo de envio de planilha.
- Fase 3:
  Admin define/aloca bolsas, cria edital, coleta assinatura e publica.
- Fase 4:
  Student se inscreve, Professor avalia/publica resultado, Student aceita.
- Fase 5:
  Admin valida dados bancarios e gera exportacoes do periodo.
- Fase 6:
  Admin inicia ciclo final, Professor/Student assinam relatorios, Admin gera
  artefatos finais e dispara envio.
